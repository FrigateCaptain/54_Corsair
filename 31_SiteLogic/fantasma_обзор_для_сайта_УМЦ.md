# Fantasma: обзор для сайта УМЦ «Ключ к реальному управлению»

## Что такое Fantasma

**Fantasma** (fantasma.io) — визуальный конструктор тем для Ghost CMS. Работает в браузере, не требует установки. На выходе — стандартный ZIP-файл темы, который загружается в Ghost Admin.

Fantasma строится поверх официальной темы **Source** (стандартная тема Ghost) и добавляет к ней лендинговые секции.

Сайт: https://www.fantasma.io  
Конструктор: https://www.fantasma.io/app

---

## Что делает и что не делает

### Делает:
- Визуальный редактор макета сайта (секции, цвета, шрифты, отступы)
- Добавляет лендинговые секции: hero-баннер, feature grid, статистика, FAQ, logo cloud
- Превью в реальном времени
- Экспорт в ZIP — стандартная Ghost-тема
- Работает с Ghost самохостингом (не только с Ghost Pro)
- **Бесплатно** (базовый функционал)

### Не делает:
- Не редактирует контент (тексты, посты, страницы) — это через Ghost Admin или скрипт
- Не имеет API — нельзя автоматизировать из Cursor
- Не сохраняет правки, сделанные вручную в `.hbs`-файлах — при повторном экспорте перезапишет

## Применимость для сайта УМЦ

### Что нужно сайту по документации:

По документу «Структура сайта» — главная страница состоит из 9 блоков с якорной навигацией:
Hero → О центре → Для кого → Что делаем → Программы → Кейсы → Книги → Каналы → Контакты

Fantasma закрывает большую часть:

| Блок главной | Секция в Fantasma | Статус |
|---|---|---|
| Hero-блок | Hero / Banner | ✅ есть |
| О центре (фото + текст) | Feature (изображение + текст) | ✅ есть |
| Для кого (портреты клиентов) | Feature Grid / Cards | ✅ есть |
| Что делаем (список услуг) | Feature Grid с иконками | ✅ есть |
| Программы (карточки) | Ghost Cards (автоматически из тегов) | ✅ есть |
| Кейсы и отзывы | Ghost Cards / пользовательский блок | ✅ частично |
| Книги | Feature Grid / Ghost Cards | ✅ есть |
| Каналы (Telegram, YouTube) | Stats / иконки со ссылками | ✅ есть |
| Контакты (форма + Telegram) | CTA-секция + HTML-блок с формой | ⚠️ форма — через HTML вручную |

### Что нужно сделать дополнительно:
- Форма записи (`/zapis/`) — вставить HTML-блок с формой (Formspree, Tally) внутри Ghost-страницы
- Тексты внутри секций — заполнять в Ghost Admin или через `ghost_publish.py`
- Блог — работает автоматически через Ghost Posts, секция Ghost Cards на главной

## Workflow: как работать с Fantasma + Cursor + Ghost

```
1. [Fantasma] → Собрать макет главной (Hero, секции, цвета, шрифты)
        ↓
2. [Fantasma] → Экспортировать ZIP
        ↓
3. [Ghost Admin] → Design → Upload theme → Активировать
        ↓
4. [Cursor / ghost_publish.py] → Публиковать контент (посты, страницы)
        ↓
5. Для изменения макета → вернуться в Fantasma, отредактировать, снова экспортировать
   Для изменения только контента → только Cursor + скрипт, Fantasma не трогать
```

### Важное правило:
> Fantasma управляет **шаблоном** (как выглядит).  
> Ghost Admin / Cursor управляет **контентом** (что написано).  
> Эти два слоя не пересекаются — можно работать независимо.

## Порядок первого запуска

1. Открыть **https://www.fantasma.io/app** в браузере
2. Выбрать шаблон: **Home** (главная)
3. Добавить секции в порядке структуры сайта УМЦ (сверху вниз):
   - Hero (заголовок + подзаголовок + 2-3 кнопки CTA)
   - Feature (О центре: фото Виталия + текст)
   - Feature Grid (Что делаем: 5-6 карточек услуг)
   - Ghost Cards (Программы: автоматически по тегу)
   - Ghost Cards (Последние посты блога)
   - CTA-секция (кнопка Записаться + контакты)
4. Настроить цвета и шрифты в соответствии с принципами:
   - Основной цвет: тёмно-синий или чёрный
   - Акцентный (кнопки CTA): один яркий цвет
   - Шрифт: sans-serif (Inter, Roboto или Open Sans)
5. Экспортировать ZIP
6. Загрузить в Ghost: **Settings → Design → Upload theme**

## Ограничения, важные для УМЦ

1. **Форма записи** — Fantasma не создаёт формы. Форму нужно вставить в отдельную Ghost-страницу `/zapis/` через HTML-блок (Formspree или Tally).

2. **Двуязычность / кириллица** — секции поддерживают кириллицу, Ghost работает с русскоязычным контентом без ограничений.

3. **Колонки внутри постов** — Fantasma управляет только страницами-шаблонами. Колонки внутри текста поста (например, для методологической статьи) — только через HTML-блок в редакторе Ghost.

4. **Нет автоматизации** — нельзя обновить тему скриптом из Cursor. Изменение макета = ручной экспорт из Fantasma + загрузка в Ghost Admin.
